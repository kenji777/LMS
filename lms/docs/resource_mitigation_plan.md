## 리소스 최적화 작업 계획

### 1. 블루투스 메시지 처리 파이프라인
- **목표**: 메시지 처리 직렬화, 캐시 활용, UI 업데이트 최소화.
- **전처리**: 처리 흐름 다이어그램, `TestData` 사용 필드 목록, UI 업데이트 지점 목록화.
- **작업**:
  1. `ExecutorService` 도입 및 `processBtPayload` 분리.
  2. 스펙 캐시(Map) 구성 → DB 조회 제거.
  3. UI 업데이트 번들화 후 단일 `runOnUiThread`.

### 2. 블루투스 재연결 제어
- **목표**: 단일 장치 우선 연결 + 백오프 재시도.
- **전처리**: 현재 타이머 주기, 목표 디바이스, 성공/실패 이벤트 파악.
- **작업**:
  1. `tmrBTList` 제거, 재시도 핸들러로 교체.
  2. 성공 시 루프 중단, 실패 시 지수백오프.
  3. 연결 상태 캐시로 중복 연결 회피.

### 3. USB 폴링 타이머
- **목표**: 필요 시에만 단일 스케줄 태스크 실행.
- **전처리**: 폴링 필요 조건 정의, 기존 생성/해제 지점 파악.
- **작업**:
  1. `ScheduledExecutorService` 기반 `start/stopUsbPolling`.
  2. 테스트/USB 이벤트에 따라 시작·중지.
  3. 무응답 시 간격 증가(backoff).

### 4. 테스트 스펙 JSON/DB 처리
- **목표**: 변경 시에만 DB 갱신 및 UI 리빌드.
- **전처리**: JSON → DTO 매핑, 현재 테이블 스키마 검토.
- **작업**:
  1. JSON diff 후 변경 없으면 skip.
  2. `TestData.bulkReplaceSpecs` 구현.
  3. UI는 요약 데이터만 업데이트.

### 5. 키오스크 모드 감시
- **목표**: 이벤트 기반 시스템 UI 숨김.
- **전처리**: 적용 대상 Android 버전 조사.
- **작업**:
  1. `OnSystemUiVisibilityChangeListener` 중심으로 재구성.
  2. `hideSystemUI()` 호출은 포커스/재개 시 1회만 수행.
